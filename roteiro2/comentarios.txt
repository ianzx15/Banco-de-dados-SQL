Questão 2:
Ao executar comandos 2 e inserir o id, foi encontrado um problema referente ao limite de caracteres permitidos
pelo tipo INTEGER, foi necessário alterar para BIGINT por meio do comando:
-> ALTER TABLE tarefas ALTER COLUMN id TYPE BIGINT;

Questão 4:
Ao alterar os atributos para não permitir valores nulos foi lançado um erro, desse modo foi necessário remover
os valores nulos que já existiam por meio do comando:
-> DELETE FROM tarefas WHERE id IS NULL;

Questão 5:
Após executados os comandos 5 foi necessário remover a segunda tarefa pelo comando:
->DELETE FROM tarefas  WHERE prioridade = 3;
Em seguida alterar o id da tarefa para PRIMARY KEY:
-> ALTER TABLE tarefas ADD PRIMARY KEY (id);

Questão 7:
Foi necessário alterar o valor da propriedade para que ela se encaixasse na nova restrição de valores entre 0 e 5
comando utilizado:
-> ALTER TABLE tarefas SET prioridade = 5 WHERE id = 9147483659;
Em seguida a nova restrição foi criada:
-> ALTER TABLE tarefas ADD CONSTRAINT prioridade_chk2 CHECK (prioridade in (0,1,2,3,4,5));
